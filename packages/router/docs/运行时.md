# Router 前端运行时

## 数据结构

1. 路由表
   ```typescript
   type RouteMap = Reocrd<
     /** key 为 浏览器 url 路径, 
     		如果是 dir/index.xxx 对应的路由是 dir 
     		如果是 dir/foo.xxx 对应的路由是 dir/foo
     **/
   	string,
     {
       /** 异步 import 组件的函数 **/
       import: () => Promise<Store>,
       /** 加载状态 **/
       status: 'idle' | 'loading' | 'loaded' | 'error',
       /** 加载中的 Promise，避免重复 import **/
       promise?: Promise<Store>,
       /** 已加载的组件实例，loaded 时有值 **/
       component?: Store,
     }
   >
   
   ```

2. 目录嵌套菜单
   ```typescript
   type Menu = {
     /** 菜单名 */
     name: string,
     /** 表示目录或文件对应的 url 路径
     	1. 文件 与 RouteMap key 相同
     	2. 目录 如果有 dir/index.xxx 那么 path 为 dir ，否则为空
     */
     path?: string,
     /** 子菜单 */
     children: Menu[],
   }
   ```

3. 不论是路由表还是，目录嵌套菜单，都可以通过 vite 和 webpack 解析得来
   ```typescript
   let modules = {};
   
   // 利用打包工具普遍支持的 process.env.NODE_ENV 或自定义变量

       modules[fullPath] = () => context(key);
     });
   }
   
   export default modules;
   ```

4. 路由器 Router

   1. Router 本身是 Store

      ```typescript
      type RouteEntry = {
        path: string;
        params: Record<string, string>;
        scroll?: number;      // 离开时的 scrollY，popstate 时恢复
        component?: Store;    // 已完成加载的组件实例
      };

      class Router extends Store {
        static [StoreIgnoreKeys] = ['map', 'menus', 'stack']

        active: RouteEntry;   // 唯一响应式出口，模板用 {active.component} 渲染
        map: RouteMap;
        menus: Menu[];
        enterGuard: (to: RouteEntry) => boolean | Promise<boolean>;
        leaveGuard: (from: RouteEntry) => boolean | Promise<boolean>;

        private stack: RouteEntry[] = [{ path: location.pathname, params: {} }];
      }
      ```

      `stack` 在 `StoreIgnoreKeys` 中——只做历史记录，不触发响应式。`active` 在 `#navigate` 中赋值，是模板唯一感知的路由变化入口。

   2. SSR → Hydrate 桥梁

      SSR 渲染时把首屏页面组件注入 `window.__INIT_ROUTE__`：

      ```html
      <!-- SSR 输出的 HTML -->
      <script type="module">
        import Page from './assets/about-abc123.js';
        window.__INIT_ROUTE__ = { '/about': Page };
      </script>
      ```

      `init()` 执行时先检查这个全局变量：

      ```typescript
      init(path: string) {
        const route = this.map[path];
        if (!route) return;

        // 优先取 SSR 注入的首屏组件（同步，无网络开销）
        const preloaded = window.__INIT_ROUTE__?.[path];
        if (preloaded) {
          route.component = preloaded;
          route.status = 'loaded';
          this.idleSet.delete(path);  // 首屏已就绪，移出预加载集合
          delete window.__INIT_ROUTE__; // 用完即删
        }

        this.active = {
          path,
          params: route.params ?? {},
          component: route.component,  // 有则同步渲染，无则由 loadComponent 异步加载
        };
      }
      ```

      SSG 场景同理——每页的 entry 中 `import Page from './assets/xxx'`，设 `routeMap['/about'].component = Page` 后调 `init`，全程同步。

   2. 历史栈

      每次 `pushState` 追加，`replaceState` 替换栈顶，popstate 时对比栈恢复。`back()`/`go()` 通过栈提前知道目标 URL，在调浏览器 API 前跑 enter 守卫。

   3. 五个公开方法 + 内部实现

      ```typescript
      // ====== 五个公开方法 ======

      pushState(url: string) {
        const target = match(url, this.map);
        if (!target) return;
        this.stack.push({ path: url, params: target.params });
        this.#navigate(target);
      }

      replaceState(url: string) {
        const target = match(url, this.map);
        if (!target) return;
        this.stack[this.stack.length - 1] = { path: url, params: target.params };
        this.#navigate(target, { replace: true });
      }

      back() {
        if (this.stack.length < 2) return;
        if (!this.checkLeaveGuard()) return;
        // 栈顶的前一个就是目标，不需要 match
        const target = this.stack[this.stack.length - 2];
        if (!(await this.checkEnterGuard(target))) return;
        history.back();  // popstate 触发 Stack pop
      }

      forward() {
        // 前进按钮：浏览器保留了 forward 历史，但 Router 栈没有
        // popstate 时检测 URL 是否匹配栈顶后一个，匹配则恢复
        history.forward();
      }

      go(delta: number) {
        if (delta < 0 && this.stack.length < -delta + 1) return;
        // 后退多步：栈中能直接定位目标
        const targetIdx = this.stack.length - 1 + delta;
        if (targetIdx >= 0 && targetIdx < this.stack.length) {
          if (!this.checkLeaveGuard()) return;
          const target = this.stack[targetIdx];
          if (!(await this.checkEnterGuard(target))) return;
        }
        history.go(delta);
      }

      // ====== 内部实现 ======

      #navigate(target: RouteEntry, { replace = false } = {}) {
        if (!this.checkLeaveGuard()) return;
        if (!(await this.checkEnterGuard(target))) return;

        // 保存当前页滚动位置
        this.stack[this.stack.length - 1].scroll = window.scrollY;

        if (!replace) {
          history.pushState(null, '', target.path);
        } else {
          history.replaceState(null, '', target.path);
        }

        const component = await this.loadComponent(target.path);
        target.component = component;
        this.active = target;  // 响应式出口，{active.component} 自动重渲染
      }

      // popstate — 浏览器前进/后退
      window.addEventListener('popstate', () => {
        const current = location.pathname;
        const idx = this.stack.findLastIndex((r) => r.path === current);

        if (idx !== -1) {
          // 在栈中 → Router 产生的历史，恢复
          this.stack.length = idx + 1;
          const entry = this.stack[idx];
          this.#navigate(entry, { guard: true });
          // 恢复滚动位置
          if (entry.scroll != null) window.scrollTo(0, entry.scroll);
        } else {
          // 不在栈中 → 外部跳转（书签、手动改 URL），重置
          this.stack = [{ path: current, params: {} }];
          this.#navigate(match(current, this.map), { guard: false });
        }
      });
      ```

      为什么 `back`/`forward`/`go` 现在也能跑完整守卫：

      | 方法 | enter 守卫 | 目标来源 |
      |------|:---:|------|
      | `pushState(url)` | ✅ `#navigate` 内执行 | 栈中新增 |
      | `replaceState(url)` | ✅ | 栈顶替换 |
      | `back()` | ✅ 栈倒数第二项 | 栈中已有 |
      | `forward()` | popstate 中执行 | 栈外恢复 |
      | `go(delta)` | ✅ delta<0 时栈中定位 | 栈中已有 |

      浏览器按钮的 `popstate` 回调中，`findLastIndex` 检查新 URL 是否在栈中——在则恢复栈顶并跑完整守卫，不在则视为外部跳转跳过守卫并重置栈。

   4. 在路由跳转时

      执行流程见上方 `#navigate`。守卫不通过时直接 return，`pushState` 未产生历史记录，`back`/`go` 未调原生方法，无需恢复。

   5. 组件异步加载

      状态存在 `RouteMap[key]` 的字段中。状态转换：`idle → loading → loaded/error`。

      ```typescript
      async loadComponent(path: string): Promise<Store> {
        const route = this.map[path];
        if (!route) throw new Error(`Route not found: ${path}`);

        switch (route.status) {
          case 'loaded':  return route.component!;
          case 'loading': return route.promise!;
          case 'error':   break; // 可重试
          case 'idle':    break;
        }

        this.idleSet.delete(path);  // 开始加载，移出待预加载集合
        route.status = 'loading';
        route.promise = route.import()
          .then((mod) => {
            const Comp = mod.default || mod;
            route.status = 'loaded';
            route.component = Comp.new();
            return route.component;
          })
          .catch((err) => {
            route.status = 'error';
            throw err;
          })
          .finally(() => {
            this.loadingCount--;
            this.preloadNext();
          });

        return route.promise;
      }
      ```

   6. 空闲预加载

      ```typescript
      class Router extends Store {
        idleSet = new Set<string>();  // 待预加载的路径
        maxPreload = 3;
        loadingCount = 0;

        initRoutes() {
          for (const path of Object.keys(this.map)) {
            if (this.map[path].status === 'idle') {
              this.idleSet.add(path);
            }
          }
        }

        preloadNext() {
          while (this.loadingCount < this.maxPreload && this.idleSet.size > 0) {
            const path = this.idleSet.values().next().value!;
            this.loadingCount++;
            const scheduler = typeof requestIdleCallback !== 'undefined'
              ? requestIdleCallback
              : (fn: () => void) => setTimeout(fn, 0);
            scheduler(() => this.loadComponent(path));
          }
        }

        afterNavigate() {
          this.preloadNext();
        }
      }
      ```

      工作方式：首屏后取最多 `maxPreload` 个并发预加载，每个完成后释放位置并启动下一个。`error` 不重试，不重新加入 `idleSet`。

## 打包方案

### Vite 插件 —— 零中间文件，虚拟模块

用户只需：

```
vite.config.ts        ← plugins: [bobeRouter({ pages: './pages' })]
pages/
  index.ts            ← export default class Home extends Store { ... }
  about.ts            ← export const meta = { title: '关于' }
```

插件内部用 Vite 虚拟模块，不产生 `.booe/gen/`：

```typescript
export function bobeRouter({ pages }: { pages: string }) {
  let routes: RouteDef[];

  return {
    name: 'bobe-router',

    config() {
      return {
        build: {
          rollupOptions: {
            input: {
              'server/entry':  'virtual:bobe-entry-ssr',
              'client/entry':  'virtual:bobe-entry-client',
            }
          }
        }
      };
    },

    buildStart() {
      routes = scanPages(pages);           // 扫描 pages/ 目录，解析 meta
    },

    resolveId(id) {
      if (id === 'virtual:bobe-entry-ssr')    return '\0bobe-entry-ssr';
      if (id === 'virtual:bobe-entry-client') return '\0bobe-entry-client';
    },

    load(id) {
      if (id === '\0bobe-entry-ssr')    return genSSREntry(routes);
      if (id === '\0bobe-entry-client') return genClientEntry(routes);
    },

    configureServer(server) {
      server.watcher.on('all', () => { routes = scanPages(pages); });
    }
  };
}
```

Rollup 从虚拟模块直接打包出 `dist/`：

```
dist/
  server/entry.js              ← SSR bundle（全量 import，Node 跑）
  client/entry.js              ← 浏览器入口
  client/about-abc123.js       ← 代码分割的页面 chunk
  about/index.html             ← SSG 生成的 HTML
```

dev 时 `load` 返回的源码不落盘，Rollup 直接内存中处理。

### 虚拟模块内容

**`virtual:bobe-entry-ssr`**（Node bundle，同步 import）：

```typescript
// 插件 load() 返回以下源码，Rollup 编译
import Home from '@/pages/index';
import About from '@/pages/about';
import Post from '@/pages/post/[id]';

export const routeMap = {
  '/':       { component: Home, meta: { title: '首页' } },
  '/about':  { component: About, meta: { title: '关于' } },
  '/post/1': { component: Post, params: { id: '1' } },
};
```

**`virtual:bobe-entry-client`**（浏览器 bundle，代码分割）：

```typescript
// 插件 load() 返回以下源码，Rollup 编译 + 代码分割
export const routeMap = {
  '/':       { import: () => import('@/pages/index'),    meta: { title: '首页' } },
  '/about':  { import: () => import('@/pages/about'),    meta: { title: '关于' } },
  '/post/1': { import: () => import('@/pages/post/[id]'), params: {} },
};

const router = createRouter(routeMap);
router.init(location.pathname);   // 先查 window.__INIT_ROUTE__
hydrate(App, document.getElementById('root'));
```

### SSR 中间件 —— 注入 window.__INIT_ROUTE__

```typescript
// 服务端代码（框架提供，引用虚拟模块的产物）
import { routeMap } from 'virtual:bobe-entry-ssr';

app.get('*', (req, res) => {
  const router = createRouter(routeMap);
  router.init(req.path);
  const html = renderHtmlStr(App);

  // 注入首屏桥接脚本，build 时 Rollup 会把 import 路径替换为实际 chunk 路径
  const initScript = `<script type="module">
    import Page from '${resolveChunk(req.path)}';
    window.__INIT_ROUTE__ = { '${req.path}': Page };
  </script>`;
  res.send(html.replace('</head>', initScript + '</head>'));
});
```

SSG 构建同理——遍历 routeMap，每页调 `router.init(path)` + `renderHtmlStr`，写入 `dist/path/index.html`。

### 用户视角

用户只写 `pages/` 目录，配一个 Vite 插件。构建产物全部进 `dist/`，无中间文件。
